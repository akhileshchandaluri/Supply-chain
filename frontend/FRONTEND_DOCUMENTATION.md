# SmartChain AI — Frontend Architecture & Documentation

This document explains the purpose and functionality of every file in the React frontend (`/frontend/src`).

## 1. Core Application Setup

*   **`main.jsx`**: The React entry point. It binds the React application to the HTML DOM and wraps it in global providers (like `BrowserRouter` for routing).
*   **`App.jsx`**: The core layout and routing controller. It defines the overall structure of the app (the sidebar on the left, the main content area on the right) and maps specific URLs (like `/forecast` or `/route`) to their respective Page components. It also manages the global state for the "Live Order" data so it can be shared across all pages.
*   **`index.css`**: Contains all global styling, modern CSS variables (glassmorphism effects, color palettes), and utility classes that give the dashboard its premium look.

## 2. API Communication

*   **`api/client.js`**: A centralized Axios or Fetch client. This file defines all the functions used to communicate with the FastAPI backend (e.g., triggering a live order, fetching historical data, or sending messages to the Explainable AI chat). Keeping API calls here keeps the UI components clean.

## 3. Reusable Components (`src/components/`)

These are small, focused UI elements used repeatedly across different pages.

*   **`Sidebar.jsx`**: The permanent left-hand navigation menu. It highlights which tab is currently active and contains the status indicator at the bottom.
*   **`MetricCard.jsx`**: A reusable, stylized card used to display key performance indicators (KPIs) like "RMSE", "Total Cost", or "Anomaly Rate". It ensures visual consistency across the dashboard.
*   **`RLInsightsPanel.jsx`**: A specialized component used in the RL Agent page. It visually breaks down the math behind the agent's decision, showing the exact Q-Values and the one-step reward calculation.
*   **`RLChatPanel.jsx`**: The chat interface widget used in the RL Agent page. It handles the message state, typing indicators, and rendering the LLM's responses when you interrogate the agent's logic.

## 4. Main Pages (`src/pages/`)

These represent the actual screens/tabs the user navigates between.

*   **`Dashboard.jsx` (Overview)**: The home screen. It displays the executive summary. It pulls data from all 5 models (Demand, Risk, Anomaly, RL Action, Route) into one unified view and displays the plain-English Explainable AI audit generated by Llama 3.
*   **`DemandForecast.jsx`**: The dedicated page for the XGBoost model. It renders the interactive line charts showing historical actuals vs. the 7-day predicted future demand, along with model accuracy metrics.
*   **`RiskAnomaly.jsx`**: Combines the Random Forest and Isolation Forest outputs. It displays the predicted delivery risk for the current order, the probabilities for Low/Medium/High, and whether the supplier has been flagged for erratic behavior.
*   **`RLAgent.jsx`**: The deepest technical page. It visualizes the current state of the supply chain (Inventory, Demand, Risk) and explains exactly why the Q-Learning Agent chose a specific action (e.g., EMERGENCY_REORDER). It houses the `RLInsightsPanel` and `RLChatPanel`.
*   **`RouteOptimization.jsx`**: The interactive geographic map. It visualizes the `A*` algorithm's output, drawing the selected path across India from the supplier to the destination warehouse, along with the total calculated logistics cost.
*   **`History.jsx`**: A data table screen. It lists every "Live Order" that has been simulated during this session, showing a log of what decisions the AI pipeline made for each consecutive run.
